Patient Flow Bottlenecks: Wait Time Drivers & Staffing Ideas
Project Overview
This project analyses patient flow data to identify bottlenecks in healthcare operations. Using diagnostic analytics, the goal is to uncover the drivers of wait times, evaluate staffing efficiency, and propose actionable improvements that enhance patient satisfaction and reduce operational delays.
•	Analytics Type: Diagnostic
•	Industry: Healthcare (Operations)
•	Dataset: Healthcare Analytics – Patient Flow Data
Objectives
•	Diagnose wait time drivers across departments and shifts.
•	Correlate staffing levels with patient satisfaction and admissions.
•	Apply decision trees/SHAP to identify high impact features.
•	Deliver what if staffing scenarios to guide operational improvements.
Key Skills & Methods
•	SQL querying (filtering, grouping, joins, correlation analysis).
•	Decision trees & SHAP feature importance.
•	DMAIC (Define, Measure, Analyse, Improve, Control) framework.
•	Dashboard design for staffing simulations.
Primary KPIs
•	Average Wait Time (minutes per patient).
•	Satisfaction Score (patient feedback).
•	Admissions Volume (per department, per shift).
Deliverables
•	SQL query bank for KPI diagnostics and feature analysis.
•	Feature importance plots (decision trees/SHAP).
•	Staffing what if dashboard (scenario simulations).
•	DMAIC summary report with actionable recommendations.
Business Impact
By targeting high impact drivers of patient wait times and optimizing staffing windows, this project aims to:
•	Reduce patient wait times.
•	Improve satisfaction scores.
•	Increase throughput of admissions.
•	Support data driven staffing decisions for healthcare operations.
